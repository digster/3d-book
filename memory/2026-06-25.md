# 2026-06-25 — Initial build of the 3D isometric book app

## What was built
Greenfield C++ app: an open book rendered in isometric view with procedurally
generated 3D models scattered across the two pages. Built on SDL3 + the SDL_GPU
API (Metal backend on macOS), GLM for math, SDL_shadercross for shaders.

## Key decisions
- **Rendering API: SDL_GPU** (not raw Vulkan/OpenGL). Chosen because the machine
  already has SDL_shadercross installed and the sibling project
  `../sdl3-2d/examples/cpp-gpu-shadercross` proves the path end-to-end.
- **Shader pipeline:** author GLSL (`shaders/scene.vert|frag`) → `glslangValidator
  -V` to SPIR-V at build time → `SDL_ShaderCross_CompileGraphicsShaderFromSPIRV`
  transpiles to MSL at load time. Avoids HLSL/DXC and raw-MSL entrypoint issues.
- **Models: procedural primitives** (box/pyramid/sphere/torus) — no asset loader,
  per user's "just place some random 3D models for now".
- **Camera:** orthographic isometric, start 45°/35.264°, mouse-drag orbit + scroll
  zoom.

## Architecture
`book_core` static lib (everything but `main.cpp`) is linked by both the app and
the headless test target. Modules: `gpu/{mesh,shader}`, `scene/{camera,book,
scene}`, `render/renderer`. The book is built from boxes (cover + two cream page
boxes whose top faces are the staging surfaces + spine); `scene/scene.cpp`'s pure
`generatePlacements()` seats random primitives on the two page rects.

## Gotchas captured (see ARCHITECTURE.md)
- SDL_GPU maps **vertex uniform buffers to descriptor set 1** → `scene.vert` uses
  `layout(set = 1, binding = 0)`. Wrong set = renders nothing.
- `GLM_FORCE_DEPTH_ZERO_TO_ONE` + `glslangValidator -V` for the [0,1] clip depth.
- Torus `minorSeg` must be a multiple of 4 so the tube's lowest point (270°) is a
  real vertex → `minY == -minorRadius` (matters for seating; caught by a test).
- The shadercross dylib lives in `/usr/local/lib`; CMake bakes an `-rpath` to its
  pkg-config `libdir` so it resolves at runtime.

## Verification (all passing)
- `cmake --build build` clean (only a benign duplicate-rpath linker warning).
- `ctest` — geometry/camera/placement unit tests pass.
- `./build/3d_book --frames N` — headless, Metal API Validation on, no errors.
- `./build/3d_book --screenshot out.bmp` — visual confirmed: book + models render
  correctly (depth, culling, orientation all good). Added `--screenshot` as a
  permission-free way to capture a deterministic frame (offscreen render +
  `SDL_SaveBMP`).

## Possible next steps
- Drag-and-drop / click-to-place models on a page (picking via ray vs page plane).
- Slight page tilt / page-curl for a more "open book" silhouette.
- Warmer/more varied color palette (current HSV roll skews cool).
- glTF/OBJ loading if real models are wanted later.

# 2026-06-25 (later) — External 3D models with textures

## What changed
Replaced the procedural primitives with a real **external model loader**. The
book now stages **OBJ and glTF 2.0 (.gltf/.glb) models with base-color textures**
loaded from a `models/` directory next to the executable. Primitives were
proof-of-concept and were removed (the book itself still uses `makeBox`).

## Key decisions
- **Libraries (vendored single-header, MIT/PD, under `third_party/`):**
  tinyobjloader (OBJ), cgltf (glTF/GLB), stb_image (PNG/JPG decode). Compiled
  once in `src/third_party_impl.cpp`; that TU is built with `-w` (CMake
  `set_source_files_properties`) so vendor warnings don't drown ours.
- **Texturing without shader branching:** the frag shader (`set = 2, binding = 0`
  sampler) always does `texture(uBaseColor, uv) * color * light`. Untextured
  objects (book parts, material-less models) bind a shared **1×1 white texture**,
  so color shows through unchanged. One draw path for everything.
- **Two-sided rendering:** pipeline `CULLMODE_NONE` + frag `gl_FrontFacing` normal
  flip, because external models have arbitrary winding. Visually identical for the
  opaque book (depth test handles it).
- **Vertex gained `vec2 uv`** (location 2, `FLOAT2`); `Instance` gained a
  `texture` handle bound per-draw with `SDL_BindGPUFragmentSamplers`.
- **Runtime model registry replaces the fixed `PrimitiveType` set.**
  `generatePlacements(left, right, seed, std::vector<ModelInfo>)` now returns
  `{modelIndex, model}`; each placement expands to one `Instance` per sub-mesh.
- **Normalization is the integration trick:** every loaded model is recentered
  (combined AABB → origin) and uniformly scaled so its largest half-extent is
  `kTargetHalfExtent` (0.4) in `io/model_loader.cpp`, yielding the same
  `{halfHeight, footprintRadius}` contract the old primitives exposed → the
  existing page seating/inset math works untouched.

## New/changed files
- New: `src/gpu/texture.*`, `src/io/model_loader.*`, `src/third_party_impl.cpp`,
  `third_party/{tiny_obj_loader,cgltf,stb_image}.h`, `models/` (samples),
  `tests/assets/` (fixtures), `tools/make_sample_models.py`.
- Changed: `gpu/mesh.h` (Vertex+uv, Instance+texture), `shaders/scene.*`,
  `gpu/shader.*` (sampler count arg), `scene/scene.*` (registry + dir load),
  `scene/book.*` (white texture on parts), `render/renderer.cpp` (3rd attr,
  no-cull, sampler bind), `CMakeLists.txt`, `tests/*`.
- Removed: pyramid/sphere/torus generators + `PrimitiveType`/`kPrimitiveInfo`/
  `makeCanonical`.

## Gotchas captured
- SDL_GPU descriptor sets: **fragment sampled textures = set 2** (vertex UBO is
  set 1). Wrong set binds the wrong resource type.
- glTF UV origin is top-left (no flip); **OBJ V must be flipped** (`v = 1 - v`).
- glTF data-URI images need base64 decode (small inline decoder); `.glb` and most
  embedded `.gltf` use a buffer-view (bytes straight from `buffer->data`).
- `SDL_GenerateMipmapsForGPUTexture` needs the texture created with **both**
  `SAMPLER | COLOR_TARGET` usage.
- `tools/make_sample_models.py` hand-builds a valid `.glb` (GLB header + JSON +
  BIN chunks, image as a buffer-view) using only the Python stdlib.

## Verification (all passing)
- `cmake --build build` clean (no warnings beyond the benign duplicate-rpath).
- `ctest` — box/camera/placement + new OBJ/glTF loader & decode tests pass.
- `--screenshot` — both sample cubes render with their textures right-side up,
  seated on the pages; book unchanged. `--frames` headless path OK.

## Possible next steps
- Richer model discovery (CLI flag, drag-and-drop) — deferred by the user.
- Optional collision-aware placement (models can overlap, as primitives did).
- Honor more glTF/OBJ material channels (normal/emissive maps) if wanted.

# 2026-06-25 (later still) — Data-driven scenes (page spreads)

## What changed
Replaced the **random** placement with **authored, data-driven scenes**. A
`book.json` file holds an ordered array of *scenes* (page spreads); each scene
places named models at specific positions on the left/right pages. The app loads
it, renders one spread at a time, and the `←`/`→` (or PgUp/PgDn) keys flip
between spreads — like turning pages. The old `generatePlacements()` RNG path,
`reseed()`, the `seed_`, and the `R` key are **gone** (user chose "replace
entirely").

## Key decisions (locked with the user, before coding)
- **Format: JSON via nlohmann/json** — vendored single header
  `third_party/json.hpp` (v3.12.0), matching the existing cgltf/stb/tinyobj
  pattern. (cJSON / a custom format were the rejected alternatives.)
- **Layout: one `book.json`** with an ordered `scenes` array (vs one file per
  scene).
- **Positioning: page-relative `(u,v)` + auto-seat** — author names a page
  (`left`/`right`) and a normalized `(u,v) ∈ [-1,1]`; the app maps it onto the
  `PageRect` (inset by footprint, clamped) and seats Y on the surface via
  `halfHeight`. The author never writes world coords or Y. (vs absolute world
  coords.)
- **Replace random entirely** (no fallback). Missing/invalid book → bare book.

## Schema (`book.json`, version 1)
`{ version:1, title?, scenes:[ { name?, objects:[ { model, page, position:[u,v],
rotation?=0(deg), scale?=1.0 } ] } ] }`. `model` is the filename as it appears in
`models/` (registry is now keyed by filename). Rotation is Y-only, degrees.

## New/changed files
- New: `third_party/json.hpp`, `src/io/scene_loader.{h,cpp}` (pure-CPU parser:
  `loadBook(path)` + `parseBook(text)` → `BookData`), root `book.json` (3 demo
  spreads using the existing `textured_cube.{obj,glb}`).
- Changed: `src/scene/scene.{h,cpp}` — removed `generatePlacements`/`reseed`/
  `seed_`; added pure `placeOnPage(page,uv,rotDeg,scale,info)`, a
  `modelByName_` map, `bookData_`/`currentScene_`, and `nextScene/prevScene/
  setScene/sceneCount/currentScene/currentSceneName`; `build()` gained a
  `scenePath` arg. `src/main.cpp` — arrow/PgUp-Dn nav, window-title feedback
  (`name (i/N)`), `--scene PATH` + `--scene-index N` flags. `CMakeLists.txt` —
  `scene_loader.cpp` in `book_core`, new `scene_assets` copy target.
  `tests/test_geometry.cpp` — `testPlacement` → `testScenePlacement` (placeOnPage
  invariants) + `testSceneLoader` (parseBook: valid/partial/malformed/version).
  README + ARCHITECTURE updated.

## Gotchas captured
- **nlohmann/json is forgiving on purpose:** `json::parse(text,nullptr,false)`
  (no exceptions) + `.value("k",default)`; a bad object is skipped (logged), a bad
  file → `nullopt` → bare book. One typo never crashes the app.
- **Registry keyed by filename** (`modelByName_`) so `book.json` can reference
  models by name; the glob name (e.g. `"textured_cube.glb"`) is the exact key.
- **`placeOnPage` clamps `(u,v)` and insets by the scaled footprint**, so an
  out-of-range or edge position still rests fully on the page (same inset math the
  old RNG used, lines 50–67 of the old scene.cpp).
- **Navigation clamps, doesn't wrap** (a real book has a first/last page).
- json.hpp include is wrapped in a `-Wfloat-equal` diagnostic-ignore guard in the
  one TU that includes it; it built clean under `-Wall -Wextra` regardless.

## Verification (all passing)
- `cmake --build build` clean (only the benign duplicate-rpath warning).
- `ctest` — box/camera + new `placeOnPage` + `parseBook` + OBJ/glTF loader tests
  pass.
- `--scene-index 0/1/2 --screenshot` — three demo spreads render distinct (verified
  by eye + differing MD5s): center-per-page, four-corners + big rotated cube, and a
  scattered row. Objects land on the named page, seated, with correct rotation/scale.
- Graceful degradation confirmed: missing book → "bare book"; unknown model
  (`ghost.obj`) → "not loaded — skipping"; `version:5` → rejected; `--scene-index
  99` → clamped. No crashes.

## Possible next steps
- Per-object full rotation (X/Z tilt) or explicit world-Y override if needed.
- Hot-reload `book.json` on a key press / file watch for faster authoring.
- A scene editor (click-to-place writing back `(u,v)` into book.json).
- Per-scene camera framing or page labels rendered on the pages.
