# 2026-06-25 — Initial build of the 3D isometric book app

## What was built
Greenfield C++ app: an open book rendered in isometric view with procedurally
generated 3D models scattered across the two pages. Built on SDL3 + the SDL_GPU
API (Metal backend on macOS), GLM for math, SDL_shadercross for shaders.

## Key decisions
- **Rendering API: SDL_GPU** (not raw Vulkan/OpenGL). Chosen because the machine
  already has SDL_shadercross installed and the sibling project
  `../sdl3-2d/examples/cpp-gpu-shadercross` proves the path end-to-end.
- **Shader pipeline:** author GLSL (`shaders/scene.vert|frag`) → `glslangValidator
  -V` to SPIR-V at build time → `SDL_ShaderCross_CompileGraphicsShaderFromSPIRV`
  transpiles to MSL at load time. Avoids HLSL/DXC and raw-MSL entrypoint issues.
- **Models: procedural primitives** (box/pyramid/sphere/torus) — no asset loader,
  per user's "just place some random 3D models for now".
- **Camera:** orthographic isometric, start 45°/35.264°, mouse-drag orbit + scroll
  zoom.

## Architecture
`book_core` static lib (everything but `main.cpp`) is linked by both the app and
the headless test target. Modules: `gpu/{mesh,shader}`, `scene/{camera,book,
scene}`, `render/renderer`. The book is built from boxes (cover + two cream page
boxes whose top faces are the staging surfaces + spine); `scene/scene.cpp`'s pure
`generatePlacements()` seats random primitives on the two page rects.

## Gotchas captured (see ARCHITECTURE.md)
- SDL_GPU maps **vertex uniform buffers to descriptor set 1** → `scene.vert` uses
  `layout(set = 1, binding = 0)`. Wrong set = renders nothing.
- `GLM_FORCE_DEPTH_ZERO_TO_ONE` + `glslangValidator -V` for the [0,1] clip depth.
- Torus `minorSeg` must be a multiple of 4 so the tube's lowest point (270°) is a
  real vertex → `minY == -minorRadius` (matters for seating; caught by a test).
- The shadercross dylib lives in `/usr/local/lib`; CMake bakes an `-rpath` to its
  pkg-config `libdir` so it resolves at runtime.

## Verification (all passing)
- `cmake --build build` clean (only a benign duplicate-rpath linker warning).
- `ctest` — geometry/camera/placement unit tests pass.
- `./build/3d_book --frames N` — headless, Metal API Validation on, no errors.
- `./build/3d_book --screenshot out.bmp` — visual confirmed: book + models render
  correctly (depth, culling, orientation all good). Added `--screenshot` as a
  permission-free way to capture a deterministic frame (offscreen render +
  `SDL_SaveBMP`).
